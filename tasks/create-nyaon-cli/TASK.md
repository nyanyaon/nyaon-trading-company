## Known plan bugs (fix while creating, not after)

The reference implementation already includes these fixes. When following the plan, apply them inline:

| Plan step | Bug | Fix |
| --- | --- | --- |
| Task 4 / `client.py:_log` | `(_LOG_DIR/f"...").open("a").write(...)` leaks FD | wrap in `with (path).open("a") as f: f.write(...)` |
| Task 6 / `test_trend.py` test 1 | calls `score(df, ema_slow=30)` but default `ema_fast=50` inverts fast/slow | call as `score(df, ema_fast=10, ema_slow=30)` |
| Task 9 / `snapshot.py:classify_diff` | benign threshold `> 0.001` rejects test's 0.04% drift | use `> 0.0001` |
| Task 12 / `mode.py:_audit_age_s` | `time.mktime` treats UTC ts as local вЂ” wrong in non-UTC zones | use `calendar.timegm(t)`; `import calendar` |
| Task 13 / `bin/nyaon` | shebang `#!/usr/bin/env -S uv run python -m nyaon_trading.cli` injects script path as argv[1] | use `#!/usr/bin/env bash\nexec uv run nyaon "$@"\n` (relies on `[project.scripts]` entry in `pyproject.toml`) |

## On failure

- Capture the failing phase number, the exact error (stderr), and the file being created.
- Write `state/incidents/<ts>.json` with `kind=create_nyaon_cli_failed`, `phase=<N>`, `error=<stderr>`.
- Escalate to user with the phase + error. Do NOT start `nyaon-cli-skill-author` or `ceo-connectivity-check` until this task passes.

## Boundaries

- Do NOT skip TDD: failing test first, observe it fail, implement, observe it pass, then commit.
- Do NOT invent flags or behaviors not in the plan.
- Do NOT modify trading logic (strategy thresholds, gate counts, risk math). The plan code is the spec.
- Do NOT run integration tests against live Binance. Testnet only (already gated by `RUN_TESTNET_TESTS=1`).
- Do NOT proceed to phase N+1 with phase N tests failing.
